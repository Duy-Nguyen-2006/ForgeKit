# UI/UX Pro Max

This is the only default UI/UX skill in ForgeKit.

## Use When

- designing or refactoring pages/components
- reviewing UI quality or UX clarity
- improving accessibility/responsiveness
- choosing layout, spacing, hierarchy, states, forms, tables, dashboards

## Do Not Use For

- pure backend/API/database work
- infra/devops tasks
- non-visual scripts

## Default Focus Order

1. accessibility
2. layout and responsive behavior
3. form clarity and interaction states
4. typography and spacing
5. performance and asset loading
6. visual polish

## Rules

- Start with the existing UI and stack.
- Prefer simple, modern, production-safe patterns.
- Keep design-system consistency.
- Avoid loading deep references unless the task needs them.
- If implementation is needed, pair with the relevant development skill instead of loading extra design skills.

## Built-In Consolidation

This skill replaces default use of:

- `frontend-design`
- `ui-styling`
- `design`
- `web-design-guidelines`
- `react-best-practices` for UI-facing concerns
